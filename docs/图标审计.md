# PodPlayer 图标审计（改造期新增 = 需你导入/统一的）

> 范围：只列**改造期间新增的 39 个**图标（修改时间 ≥ 2026-06-06）；原项目自带的 35 个（克隆时刻 06-05 16:58）不在此列、保持不动。

> 动作图例：**[换]** 换成 Lucide 对应图标统一描边；**[补绘]** 库里没有、Claude 按同规范手绘；**[加变体]** 需要"未/已"状态对；**[删]** 孤儿无引用、不用导入；**[留]** 你满意的自绘，保留(仅核对 2px 对齐)。

## 四、应成对的"状态变体"（成熟 UI 的关键，逐项核对）

| 动作 | 未态 | 已态 | 现状 |

| 收藏 | `heart`(原自带) | `heart-solid`(原自带) | ✅ 原项目已有一对 |
| **加入播放列表** | `list-plus`(现 layer-plus) | **缺** | ❌ **要补**——这就是你说的"加入后还是原图标" |

→ 统一到 Lucide 后，"已态"若库内无 filled 版，**用同一 path 填充 `fill:currentColor` 即可**（Claude 一行生成），保持同一视觉语言。

## 五、落地方式（交 Code 模式批量执行）

1. **定基准**：Lucide outline，24/2px/round/`currentColor`，命名约定 `xxx` / `xxx-active`。
2. **批量换**：上表 [换] 项，从 lucide.dev 取对应 svg 覆盖 `src/assets/icons/`（同名即可，引用处不用动）。
3. **补变体**：`layer-plus` 加 `list-check` 已加入态 + 接入"在列表中"判断。
4. **补绘自定义**：`on-air-square` / `router-wifi-alt` / `wifi` 你满意，保留；若描边非 2px，Claude 按基准重绘对齐。
5. **删孤儿**：第二节清单（先确认 seek-15/30）。
6. **换丑的**：`social-network` → `bookmark`/`flag`。

> 统一后所有图标走 `currentColor` → 自动适配深浅色 + 激活态变色（如收藏变红、已加入变绿），不用再为每个状态单独存文件。
