# PodPlayer 图标动效规范（v3 · 定稿方向）

> 演进：用户审完 v2 全量动效后，**只保留 6 个动效**（标记 / 删除 / 节目 / 清理 / 下载 / 收藏），其余图标**统一静态**。本文据此收敛。配套可视化见对话内 `icon_set_v3_final_6anim_rest_static` 演示。
> 原则（用户提炼）：动作=图标功能的可视隐喻；只动承载语义的部件、其余钉死；从原位出发、动完回原位；动效宁缺毋滥。

## 一、核心原则（5 条）

1. **默认静态**：绝大多数图标不动。只有"动作语义强、且能提升信息量"的少数才配微动效（本版定为 6 个）。
2. **动件 / 锚件分离**：只动承载语义的部件，其余作静止锚点（托盘、桶身、基线、中心点不动）。
3. **原位闭环**：从初始态出发、回到初始态（往复位移 / scale 1→k→1 / opacity 循环），无限循环不跳变。
4. **只用合成器属性**：`transform` / `opacity`；`transform-box:fill-box` + `transform-origin` 锚到动件；不碰 box-shadow/blur → 零负担无 bug。
5. **无障碍**：`@media (prefers-reduced-motion: reduce)` 时全部静态。

## 二、保留动效的 6 个（仅此 6 个会动）

| 图标 | 动件（动） | 锚件（不动） | 动作 | 实现 | 触发 |
|---|---|---|---|---|---|
| 下载 | 箭头+杆 | 托盘 | 上下跳 | translateY 往复 | 下载中·常驻循环 |
| 节目 | 两侧信号弧 | 中心圆点 | 由内向外扩散 | opacity 阶梯（内先亮） | 在播·常驻；否则静态 |
| 标记(书签) | 书签 | — | 轻下沉（像盖章） | translateY 微 | 点击标记·触发一次 |
| 删除 | 桶盖+把手 | 桶身 | 掀盖 | rotate −24°，origin left-center | 点击·触发一次 |
| 清理 | 橡皮擦 | 底部基线 | 左右擦动 | translateX 往复 | 点击·触发一次 |
| 收藏 | 整颗心 | — | 心跳 + 空心↔红实心 | scale beat + fill 切换 | 点击·触发一次 |

> 触发：下载/节目是"持续状态"→常驻循环；其余 4 个是"一次性操作"→默认不加 animation，事件时 JS 加 `.play` class 跑一次、`animationend` 移除，不常驻。

## 三、其余图标：统一静态（不动）

队列（去音符，列表+播放三角）、睡眠、排序、插播、订阅、已订阅、取消订阅、时长、发现、屏蔽、刷新、历史、进入、加入列表（未/已）、已完成、收藏（未）。

- 统一规格：`viewBox 0 0 24 24` / `stroke-width:2` / round cap+join / `currentColor`。
- 状态变体成对，颜色用状态色：**已订阅 / 已加入 / 已完成 = `--color-success`(绿)**，**收藏(已) = `--color-danger`(红)**；其余跟随 `currentColor` 自动适配深浅色。
- 加入列表：上下两条长横夹住中间符号，未加入 `+` / 已加入 `✓`(绿)。
- 收藏：未=空心描边 / 已=红实心（实心才是已收藏，之前显黑就是因为默认给了实心）。

## 四、保留不重绘（你满意 / 趣味语义，照旧）

- NAS：`router-wifi-alt`（导航栏状态）/ `wifi`（单集标识）——机器+wifi 自绘，你满意。
- 品牌 `on-air-square`（logo）。
- **首页"再来一瓶"= `bottle-cap`（瓶盖）**——呼应"再来一瓶"文案的趣味，**保留**，不换 Lucide。

这三类是有意的自定义/趣味设计，不在重绘范围。

## 五、实现要点（交 Code 模式）

- 动效仅 6 个，CSS keyframes 见演示；持续状态常驻，一次性走 `.play` 触发。
- 静态图标：无 `animation`，纯 path。
- 全局：`@media (prefers-reduced-motion: reduce){ [class^=ico-anim]{animation:none!important} }`。
- 复杂/趣味图标（第四节）保留现状，本规范只覆盖线性图标。
